Authentication API Reference

This guide covers OAuth 2.0 authentication for our REST API.

Getting Started

To authenticate, send a POST request to /auth/token with your credentials.

curl -X POST https://api.example.com/auth/token \
  -H "Content-Type: application/json" \
  -d '{"client_id": "YOUR_ID", "client_secret": "YOUR_SECRET"}'
Note: Always store credentials securely and never expose them in client-side code.

Token Refresh

Tokens expire after 1 hour. Use the refresh token to obtain a new access token.

The refresh flow prevents users from having to re-authenticate frequently.

curl -X POST https://api.example.com/auth/refresh \
  -H "Content-Type: application/json" \
  -d '{"refresh_token": "YOUR_REFRESH_TOKEN"}'

Error Handling

The API returns standard HTTP status codes. Common errors:

  • 401 - Invalid or expired credentials
  • 403 - Insufficient permissions
  • 429 - Rate limit exceeded
